Two Pointers Problems

「双指针问题」英文里更多时候被称为「Two Pointers Problems」。相对于和Dynamic Programming、Greedy、Backtracking等，似乎没什么人把「Two Pointers」称之为一种算法。根据我的理解，「双指针」处理指的是一种常见的解题思路，它可以用来解决一类问题。基于它的Solution常常有如下特点：

  • 时间复杂度为O(n)；
  • 常常结合一维DP（Dynamic Programming）思路；

LeetCode对Two Pointers题目进行了归类，迄今为止，除了两道hard级别题目以及需要付费完成的题目之外，我提供了所有问题的Solution并被Accpeted。

「Two Pointers」问题一般处理的对象是数组或者字符串。所谓「双指针」，指的是在处理过程中使用两个指针，对这两个指针根据一定逻辑进行移动，进而能够求得问题的解。经过个人总结，以为双指针的移动策略有两种：「夹逼」和「追逐」。

P.S：「Two Pointers」问题处理的对象也有可能是链表，一般是「双指针追逐」问题，譬如「判断链表是否有环」问题。

所谓「夹逼」指的是两个指针在刚开始时分别置于数组（或字符串）的头部或尾部，然后向内侧根据具体的策略进行移动…

所谓「追逐」指的是两个指针在刚开始时均置于数组（或字符串）的头部，然后向后根据一定策略进行移动…

下文将分别以「夹逼」和「追逐」为类别在LeetCode中挑选几道典型的「Two Pointers」问题进行简单介绍。

夹逼

「夹逼」两个字怎么看都感觉很邪恶，但叫惯了感觉还是不错^_^

如上文所言，所谓「夹逼」指的是两个指针在刚开始时分别置于数组（或字符串）的头部或尾部，然后向内侧根据具体的策略进行移动，以下是一些应用场景。

Two Sums

LeetCode中关于「Two Sums」问题有一个系列（好几道），这里指的是Two Sum II，本题属于收费内容，虽然不能提交代码参与LeetCode测试，但是仍然能够在网上看到该题目的描述：

Given an array of integers that is already sorted in ascending order, find two numbers such that they add up to a specific target number.
 
The function twoSum should return indices of the two numbers such that they add up to the target, where index1 must be less than index2. Please note that your returned answers (both index1 and index2) are not zero-based.
 
You may assume that each input would have exactly one solution.
 
Input: numbers={2, 7, 11, 15}, target=9
Output: index1=1, index2=2

该题或许是阐述「Two Pointers」最好的例子了。简单阐述一下思路：初始化左指针left指向数组起始，初始化右指针right指向数组结尾，根据已排序这个特性：

  • 如果numbers[left]与numbers[right]的和小于target，说明应该增加和，因此left右移一个较大的值；
  • 如果numbers[left]与numbers[right]的和大于target，说明应该减小和，因此right左移一个较小的值；
  • 如果numbers[left]与numbers[right]的和等于target，直接返回left和right；

此思路的时间复杂度O(n)，one-pass，空间复杂度O(1)。

P.S：暴力求解的思路是比较所有两个元素，瞧瞧它们的和是否等于target，根据高中的数学知识，容易知道需要计算C2n次；以上的这种思路是否可能会错过某些「组合」呢？确实会过滤掉许多组合，但是这种过滤是无害的，笔者曾在这篇博客中阐述过缘由。

追逐

Minimum Size Subarray Sum

Minimum Size Subarray Sum为「双指针追逐」问题提供了一个非常棒的场景描述：

Given an array of n positive integers and a positive integer s, find the minimal length of a subarray of which the sum ≥ s. If there isn’t one, return 0 instead.
 
For example, given the array [2,3,1,2,4,3] and s = 7,
the subarray [4,3] has the minimal length under the problem constraint.

解题思路：两个指针begin和end分别从数组0位置开始「启程」，end向后移动，当子数组[begin,end]的元素和大于等于target时，计算end-begin+1，此时begin开始向后移动，直到子数组[begin,end]元素和重新小于target。
