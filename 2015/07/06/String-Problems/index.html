罗马数字和正整数

罗马数字和正整数之间如何转换？如果没有遇到这道题，还真不会。只隐约记得小学老师教过，但完全忘了。

参考百度百科罗马数字。罗马数字由7个字符组成，这7个字符分别对应不同的整型值：I-1，V-5，X-10，L-50，C-100，D-500，M-1000。

不同的组合构建了不同的值，组数规则很简单：

  1. 相同的数字连写，所表示的数等于这些数字相加得到的数，如，III = 3；
  2. 小的数字在大的数字的左边，所表示的数等于大数减小数得到的数，如，IV = 4，IX = 9；
  3. 小的数字在大的数字的右边，所表示的数等于这些数字相加得到的数，如，VII = 8, XII = 12；

说明：

  • 规则2优先于规则3；
  • 4和9以及相关数（譬如40，90等）要求使用规则2表示，即不能用IIII表示4，而该用IV表示。

P.S：罗马数字表示法还有一些其他规则，本文不需要了解，就此略过。

罗马数字表示较小的正整数尚可，但表示比较大的数就很困难。

最长子串问题

所谓最长子串问题的一般形式是：求满足某种条件的最长子串。

Longest Substring Without Repeating Characters

LeetCode有题：Longest Substring Without Repeating Characters，描述如下：

Given a string, find the length of the longest substring without repeating characters. For example, the longest substring without repeating letters for “abcabcbb” is “abc”, which the length is 3. For “bbbbb” the longest substring is “b”, with the length of 1.

简单来说，本题要求中找出最长子串，该子串满足的条件：不含重复字符。

我的解题思路：将本题当做双指针（Two Pointers）问题处理，已知字符串s，使用两个指针（begin和end）记录满足要求的子串，则意味着s[begin, end]子串中不含任何重复字符，同时，使用一个map（或曰「字典」）记录这些字符的索引位置；end向前推进，当s[end+1]被包含在map中时，意味着s[begin, end]中有与s[end+1]相同的字符；此时的任务是找出与s[end+1]相同字符的所以位置j，并将[begin, j]中的所有字符从map中删去，begin则重新定位到j+1，s[begin, end]此时又变成了符合要求的子串了。

Interleaving String

LeetCode有题：Interleaving String，描述如下：

Given s1, s2, s3, find whether s3 is formed by the interleaving of s1 and s2.
 
For example,
Given:
s1 = "aabcc",
s2 = "dbbca",
 
When s3 = "aadbbcbcac", return true.
When s3 = "aadbbbaccc", return false.

不难判断这是一道二维DP问题，我的分析过程如下：
s1(i)表示s1的前i个字符子串，s2(j)表示s2的前j个字符子串，i和j都从1开始计；s3(i+j)表示s3的前i+j个字符子串；
而s1[i]，s2[j]，s3[i+j]分别表示s1、s2、s3的第i、j、i+j个字符；
p(i,j)记录s3(i+j)是否是s(i)和s(j)的Interleaving String，若是，则p(i,j)=True，否则p(i,j)=False。则很容易得到状态转换式，站在p(i,j)的角度，对于：

  1. s1[i]==s3[i+j]
  2. s2[j]==s3[i+j]

若1和2同时满足，则p(i,j)=p(i-1,j) || p(i,j-1)；
若只满足1，则p(i,j)=p(i-1,j)；
若只满足2，则p(i,j)=p(i,j-1)；
若1和2都不满足，则p(i,j)=False；

得到了状态方程，余下的都是细节问题了，代码详见这里。
