For example, given candidate set 2,3,6,7 and target 7,
A solution set is: [7]，[2, 2, 3]

Combination Sum II

题目描述如下：

Given a collection of candidate numbers (C) and a target number (T), find all unique combinations in C where the candidate numbers sums to T.
 
Each number in C may only be used once in the combination.
 
Note:

  • All numbers (including target) will be positive integers.
  • Elements in a combination (a1, a2, … , ak) must be in non-descending order. (ie, a1 ≤ a2 ≤ … ≤ ak).
  • The solution set must not contain duplicate combinations.

For example, given candidate set 10,1,2,7,6,1,5 and target 8,
A solution set is: [1, 7]，[1, 2, 5]，[2, 6]，[1, 1, 6]

和上一道题不太一样，每个元素仅能使用一次。解法几乎一样，唯一需要注意的是「避免重复解」。如何避免重复呢？一般的做法是增加约束：如果两个数相同，我们先用前一个数，只有当前一个数用了，这个数才能使用。
譬如对于1 1，基于我们的约束，当要使用第二个1的时候，前提是第一个1已经被使用了，这样就能够避免[1]重复。

Array and Sum系列

Array and Sum系列问题和上文的Combination Sum系列问题有些类似，都是处理数组子集的sum问题，不同的是Array and Sum系列问题的约束更多，这意味着可以用更好的策略解决问题。

Two Sum

LeetCode有题：Two Sum，描述如下：

Given an array of integers, find two numbers such that they add up to a specific target number.
 
The function twoSum should return indices of the two numbers such that they add up to the target, where index1 must be less than index2. Please note that your returned answers (both index1 and index2) are not zero-based.
 
You may assume that each input would have exactly one solution.
 
Input: numbers={2, 7, 11, 15}, target=9
Output: index1=1, index2=2

虽然题目所给的tags包括关键字「Hash Table」，意味着你可以使用散列表解决问题。但我在处理算法问题时向来不喜欢使用高级数据结构，因此笔者把它当成一道综合题（有多个过程组成）解决：

  1. 对数组进行排序，并记录排序前各个元素的位置；
  2. Two Pointers处理；

具体的「Two Pointers」处理指的是：初始化左指针left指向数组起始，初始化右指针right指向数组结尾，对于已排序数组：

  1. 若nums[left]和nums[right]的和sums小于target，说明应该增加sums，因此left右移指向一个较大的值；
  2. 若nums[left]和nums[right]的和sums大于target，说明应该减小sums，因此right左移指向一个较小的值；
  3. 若nums[left]和nums[right]的和sums等于target，game over！

排序处理的时间复杂度为O(nlgn)，Two Pointers处理时间复杂度为O(n)，总体复杂度为O(nlgn)。Solution代码比较长，详见我的github。

3Sum

LeetCode有题：3Sum，描述如下：

Given an array S of n integers, are there elements a, b, c in S such that a + b + c = 0? Find all unique triplets in the array which gives the sum of zero.
 
Note:

  • Elements in a triplet (a,b,c) must be in non-descending order. (ie, a ≤ b ≤ c)
  • The solution set must not contain duplicate triplets.

For example, given array S = {-1 0 1 2 -1 -4},
A solution set is: (-1, 0, 1)，(-1, -1, 2)

这仍然是K Sum Problem。暴力解法是每个人都能想到的，三层for循环，时间复杂度为O(n3)。

存在O(n2)的Solution。解决问题方法很直接，先排序，然后对数组遍历，挨个取出每个元素后，基于后面的元素退化处理Two Sums Problem，我的Solution代码详见github.

3Sum Closest

LeetCode有题：3Sum Closest，描述如下：

Given an array S of n integers, find three integers in S such that the sum is closest to a given number, target. Return the sum of the three integers. You may assume that each input would have exactly one solution.

For example, given array S = {-1 2 1 -4}, and target = 1.
The sum that is closest to the target is 2. (-1 + 2 + 1 = 2).

相对于上一道题，这道题的题意虽然更复杂一些，但其实更简单一些。思路很明了：先排序，然后左右夹逼，复杂度O(n2)。

4Sum

LeetCode有题：4Sum，描述如下：

Given an array S of n integers, are there elements a, b, c, and d in S such that a + b + c + d = target? Find all unique quadruplets in the array which gives the sum of target.
 
Note:

  • Elements in a quadruplet (a,b,c,d) must be in non-descending order. (ie, a ≤ b ≤ c ≤ d)
  • The solution set must not contain duplicate quadruplets.

For example, given array S = {1 0 -1 0 -2 2}, and target = 0.
A solution set is:
(-1, 0, 0, 1)
(-2, -1, 1, 2)
(-2, 0, 0, 2)

和3Sum问题一样，先排序，然后左右夹逼，复杂度O(n3)，代码详见github。不被Accepted，应该是时间复杂度太高了吧，要想降低时间复杂度最直接的办法就是「空间换时间」了。
