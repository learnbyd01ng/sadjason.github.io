写在前面

和数组一样，链表是一种非常基础的数据结构，链表包括单向链表（简称「单链表」）、双向链表以及循环链表。围绕链表结构有很多操作，譬如增加/修改/删除/查找节点，旋转链表，反转链表，对链表进行排序，合并多个单链表等等。

LeetCode中有二十多道与链表相关的算法题，这些算法题在LeetCode中的难度中等偏下，笔者刷完了所有LeetCode中关于linked list的题目（大部分原创，因为愚钝，有些题实在无法找到被能够Accepted的答案进而参考了一些资料），本文算是对这些算法题的一些总结。

P.S：当然，本文持续更新，可能也会包括一些来自于其他地方（LeetCode之外）的题目。

本文的算法实现代码全部使用Python，使用Python实现算法的愉悦感是C/C++等语言无法比拟的！

判断链表是否存在环

LeetCode有题：Linked List Cycle，描述如下：

Given a linked list, determine if it has a cycle in it.
 
Follow up:
Can you solve it without using extra space?

笔者愚钝，想了很久都没想到解决方案，后来只能在晚上找答案，几乎所有答案都是一致的：
设置两个指针fast和slow，初始值都指向链表头，slow指针每次前进1步，fast每次前进2步，如果链表存在环，则fast必定先进入环，而slow后进入环，之后两个指针指针必定相遇，否则两个指针必定会遍历完链表。

为什么fast一定能追上slow呢？结合下图简单分析一下。

在环内，fast总会逼近slow，当fast快追上slow时（t时刻）会有上图两种情况：相对位移分别为1和2：

  • 当t时刻fast和slow相对位移为1时，t+1时刻fast的位置对应3+2=5，slow的位置也对应4+1=5，意味着t+1时刻，fast和slow重逢；
  • 当t时刻fast和slow相对位移为2时，t+1时刻fast的位置对应4，slow的位置对应5，相对位移转为1了，t+2时刻二者重合；

P.S：fast和slow的行进速度分别为2和1，这种设定满巧妙的！

定位环的开始位置

LeetCode有题：Linked List Cycle II，描述如下：

Given a linked list, return the node where the cycle begins. If there is no cycle, return null.
 
Follow up:
Can you solve it without using extra space?

问题的具体描述是，若链表存在环，则找到这个环的开始位置。

这个问题比「判断链表是否存在环」要复杂一些。我仍然无法凭一己之力解决这个问题，求助于网络，所有答案几乎一致；下面将结合下图来阐述Solution。

依然定义fast和slow两个指针，二者仍然从head出发（对应上图H点），分别以2和1的速度行进，上图将环的起点定义为S，依然是fast指针先进入环（顺时针），之后与slow在X点相遇。

如上图所示，设H-S的距离为m，环的长度为n，fast和slow相遇的点X与S相对位移为x。假设fast和slow指针第一次相遇时fast已经在环内游走k圈（k>=0），依据fast的速度等于slow速度的两倍，所以我们很容易得出这样的等式：2(m + x) = m + x + kn。

P.S：左式中的m+x表示slow在第一次在与fast相遇时所走的距离，为什么是m+x呢？而不是m+x+pn(p为自然数)呢？自己想吧，很容易想明白！

2(m + x) = m + x + kn等价于m = kn-x。假设fast和slow第一次相遇的时刻为t，此时若在t时刻将slow指针重新定位到链表头（即H位置），fast位置不变（在X位置），之后让它们继续行进，只是二者的速度都为1…为什么要这样做呢？

t时刻，fast在X位置，slow重新移到在H位置，slow再次到达S位置的时间为m/1=m，此时fast在哪儿呢？fast在此m时间内走的距离为m（根据上述等式，m=kn-x），之前离S的距离为x，所以fast此时的位置为S！而S就是我们需要找到的node。

OK，到了这里，处理逻辑比较清晰明了：

  • 定义两个指针fast和slow；
  • 0时刻，fast和slow均从head出发，分别以2和1的速度行进；
  • 某时刻，fast进入环，在环内游走，slow继续龟速前进；
  • t时刻，fast和slow在环内第一次相遇，此时将slow重新定位到head，fast保持不变，然后二者均以速度1行进；
  • 某时刻，fast和slow相遇，所在位置即为目标结点。

Reorder List

LeetCode有题：Reorder List，描述如下：

Given a singly linked list L: L0→L1→…→Ln-1→Ln,
reorder it to: L0→Ln→L1→Ln-1→L2→Ln-2→…
 
You must do this in-place without altering the nodes’ values.
 
For example,
Given {1,2,3,4}, reorder it to {1,4,2,3}.

简单介绍这道题的思路：

  • 将原链表一截为二：
    • l1: L(1)->L(2)->L(2)->…->L((n+1)/2-1)
    • l2: L((n+1)/2)->…->L(n-2)->L(n-1)->L(n)
  • 反转l2；
  • 合并l1和l2
