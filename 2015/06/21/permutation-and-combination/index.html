写在前面

排列和排序完全是两码事儿！

在刷LeetCode题目过程中，发现有几道题的出发点比较类似，可以说属于一种类型，而这种类型的题目非常常见，因此觉得有必要写一篇博客对它们进行稍微地总结。

根据我的理解，本文所要分析的算法题目属于高中数学知识「排列组合」的程序化衍生，包括如下算法题：

  • 全排列：Permutations和Permutations II；
  • 全组合：Subsets和Subsets II；
  • 部分组合（m个元素的组合）：Combinations；

P.S：以上这些算法题目在LeetCode中都被归并到backtracking这个category中。
P.S：日后或许会有补充。

字典序法

直到看到上文提到的这篇博客《算法之排列与组合算法》，才意识到排序问题不仅仅是是LeetCode孤立的一道题目，它是非常常见的数组问题。关于这个问题的最常见处理方法被称为「字典序法」。

下面将对字典序法展开介绍。

首先看什么叫字典序，顾名思义就是按照字典的顺序（a-z, 1-9）。以字典序为基础，我们可以得出任意两个数字串的大小。比如 1<12<13。 就是按每个数字位逐个比较的结果。对于一个数字串123456789， 可以知道最小的串是从小到大的有序串123456789，而最大的串是从大到小的有序串987654321。这样对于123456789的所有排列，将他们排序，即可以得到按照字典序排序的所有排列的有序集合。

字典序法要解决的问题是：已知当前序，求下一个字典序。譬如某个当前序12345，不需要太多的数学知识，一眼就能看出下一个序是12354，站在程序代码的角度，如何解决这个问题呢？

很多的博客上都详细介绍了字典排序的具体操作过程，本文也没啥特别的说明，直接摘录整合。

设P是一个长度为n序列，pi表示第i个元素，则P=p1p2……pn，在「字典序法」算法中，我们要处理的事情包括：

  1. 从排列的右端开始，找出第一个比右邻数字小的数字的序号j，即j=max{i|pi<pi+1}，譬如对于3465，从第3个元素开始，6右邻数字5大，继续向左推进，4比右邻6小，满足要求，所以j对应4的序号；
  2. 在pj的右边的数字中，找出所有比pj大的数中的最小数字pk，譬如j对应数字4，4的右侧数字包括6和5，比4大的最小数字是5，所以k对应5的序号；
  3. 交换pj和pk的值；
  4. 反转序号j之后的数字；

全组合

有排列，必然有组合嘛！和排列问题一样，组合也是一种常见（却不经典）数组算法问题。

严格来讲，组合问题可以区分为「全组合」」和「部分组合」」，所谓全组合指的是n个元素的所有可能组合；非全组合指的是n个给定元素，从中选择m个元素组成一个集合，求所有组合方式。

本节内容主要针对全组合问题，非全组合问题下一节会有涉及。

当集合不含重复元素时

n个不同元素有多少种组合？若对高中数学还有一点印象，想必还记得如下公式：
C0+C1+C2+…+Cn = 2n

P.S：表达式中Ci应该写成Cin，只是在博客中不方便使用HTML标签描述该数学公式…你明白就好。

简单来说，n个不同元素一共有2n种组合方式。

LeetCode有题：Subsets，描述如下：

Given a set of distinct integers, nums, return all possible subsets.
 
Note:

  • Elements in a subset must be in non-descending order.
  • The solution set must not contain duplicate subsets.

可以看到，有些路径得到的组合会有重复，对于[7,7]，不同的路径可能会对应相同的组合：

  • (0,0) -> []
  • (0,1) -> [7]

若nums[i]==nums[i+1]，且aux[i]==True，则aux[i+1]必须为True（即将左枝叶给砍掉）。
