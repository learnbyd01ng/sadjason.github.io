写在前面

leetcode中有一道题：

Insertion Sort List

Sort a linked list using insertion sort.

题意非常简单：采用插入排序的防护四对一个链表进行排序。插入排序算是除了冒泡排序之外最简单的排序方式了，为了解决这个问题，顺便复习一下插入排序相关的知识，遂成此文。

插入排序简单介绍

引用《算法导论的描述》：

插入排序的工作方式像许多排序一手扑克牌。开始时，我们的左手为空并且桌子上的牌面向下。然后，我们每次从桌上拿走一张牌并将它插入左手中正确的位置。为了找到一张牌的正确位置，我们从右到左将它与在手中的每张牌进行比较，如下图所示。拿在左上的牌总是排好序的，原来这些牌是桌子上牌堆中顶部的牌。

简单分析一下，插入排序是一个非常不稳定的排序算法，换句话说，其时间复杂度是不太靠谱的，最好情况下，当然是所有元素已经排好了，时间复杂度为O(n)；最坏情况下，原始元素已逆向排好序，这种情况下，插入排序的时间复杂度和冒泡排序是一样一样的，即O(n2)。插入排序不是一个好的排序算法的更重要原因是“其平均情况往往和最坏情况大致一样差”。

此外，插入排序是原址排序，这算其一大优点吧，另外一个优点就是“实现极其简单”。

上述代码原理是，在已排序好的链表中插入一个新的node，从链表的头部开始挨个儿比较，如果发现某个node的val值比新的node的val值大，就将新的node插入到该node的前面。这个原理没有啥问题，但可以做得更好，不要从head开始比较，而是第一次与tail比较，即将新node与其前驱比较，如果新node的val值比前驱大，就根本没必要从head开始挨个儿与链表进行比较；做了这点修改之后，代码便通过了leetcode的测试。
